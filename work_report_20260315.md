# Tucson Link 專案工作與功能總結報告 (2026-03-15)

本報告總結了近期針對儀表板系統進行的 UI 優化、數據格式重構、硬體網關配置以及系統穩定性提升之工作成果。

---

## 1. 儀表板 UI/UX 優化 (`assets/cd.html`)
針對駕駛視覺體驗進行了深度微調，強化了資訊的可讀性與美感。

*   **新增資訊卡片**：在動態區塊右側成功整合了「里程 (ODO)」與「油箱」並排顯示卡片。
*   **標籤風格演進**：
    *   經過多輪圖示測試，最終依照使用者回饋恢復為「文字標籤」顯示（`ODO` 與 `油箱`），視覺更為穩重。
    *   採用 `items-baseline` 對齊，確保文字標籤與大體積數據數值在視覺基準線上保持一致。
*   **數據精簡化**：里程與油箱數值全面改為**整數顯示**（四捨五入），消除不必要的小數位數。
*   **警示系統升級**：
    *   **低油位警告**：新增當油量低於 **15%** 時，數值觸發紅色呼吸燈特效（`alert-pulse`）。
    *   **視覺一致性**：警示特效風格與原有的胎壓異常、水溫過熱保持完全同步。
*   **版面平衡**：微調了車速、時間與 TPMS 卡片的邊距（Margin），解決了視覺重心偏左的問題。

## 2. 數據傳輸協議重構 (WebSocket JSON)
為了對接後端系統並提升結構化程度，全面重構了 WebSocket 的傳送格式。

*   **標準化結構**：定義並實作了全新的 JSON 規範：
    ```json
    {
      "_type": "location",
      "tid": "obd",
      "fuel": 50,
      "mileage": 26500,
      "tires": {
        "fl": 33, "fr": 34, "rl": 35, "rr": 36
      },
      "speed": 80,
      "rpm": 1200,
      "temperature": 85,
      "battery": 60.5
    }
    ```
*   **同步修改範圍**：
    *   **App 發送端**：更新了 `dashboard_screen.dart` 的即時傳送與定期同步邏輯。
    *   **設定測試端**：更新了 `settings_screen.dart` 的測試按鈕數據。
    *   **網頁接收端**：更新了 `cd.html` 的解析腳本，支援「嵌套式 (Nested)」胎壓物件解析，並保持對模擬器的向下相容。

## 3. ESP32 網關韌體更新 (`esp32_relay_gateway`)
完成了硬體端的配置更新與燒錄流程優化。

*   **環境配置**：
    *   更新 Wi-Fi SSID 為 `opposky`。
    *   更新 MQTT 通訊主題 (Topic) 為 `owntracks/mt/obd`。
*   **上傳參數優化**：
    *   診斷並解決了高倍速燒錄不穩定的問題。
    *   成功切換至穩定的 **`460800`** baud 進行燒錄，確保韌體完整寫入。

## 4. 系統邏輯與連線穩定性
針對車用環境的複雜性，加入了多項自動化連動邏輯。

*   **WiFi 自動確保機制**：整合 `wifi_iot` 套件，在 App 充電且準備開啟 WebSocket 前，會自動檢核並確保手機已連上 `nx4_obd_relay` 基地台。
*   **數據傳送門檻 (Safety Check)**：
    *   **關鍵功能**：新增「OBD 未連線不傳送」邏輯。
    *   **機制**：僅在 App 成功與車輛 OBD 藍牙握手且狀態為 `connected` 時，才會啟動遠端數據發送，有效過濾無效數據與空值。

## 5. 開發與測試工具 (`simulator.html`)
同步更新了網頁模擬器，支援所有新功能測試：
*   新增里程滑桿與油位滑桿（包含 15% 警示門檻模擬）。
*   情境按鈕優化，一鍵模擬所有異常狀態（超速、低胎壓、過熱、低油位）。
